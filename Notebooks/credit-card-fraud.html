Credit Card Fraud Detection¶

In this project, the goal is to come up with a binary classification model to detect whether a credit card transaction is likely to be fraud or not.

This project will focus on data cleaning and model building. To approach the missing data, we will observe the percentage of missing data of each feature and choose different methods and handling such data based on the volume that is missing. We will also compare various binary classification models, mainly using the F1 score as a metric to filter out the best model to use for this problem. We will compare the following models:

  • K-Nearest Neighbors
  • Support Vector Machine
  • Logistic Regression
  • Decision Tree

SMOTE and Random Undersampling techniques are used to account for the unbalanced dataset to allow for better quality training data and to ensure the results produced by the models are accurate. Furthermore, we will use an encoder to process non-numeric data so that the models can read and process the information.

Summary of Results¶

After comparing various classification models, the best model for this problem is the Random Forest Classifier, which can successfully classify a credit card transaction with an F1 score of 0.8.

For more details on how the model is built, please refer to the full project below.

Dataset¶

To avoid time constraints for obtaining a large volume of data and confidentiality issues due to the nature of credit card transactions, we will use an existing dataset for this project's purpose. The data is publicly available on Kaggle, containing over 300,000 credit card transactions with over 100 features.

In this project, we will use two datasets, the application data and the previous application dataset. We will combine these two datasets by the transaction ID and use the combined dataset for exploration and model training.

Data Cleaning¶

After observation, some of the entries are XNA. These values will be replaced with NA values in the DataFrame.

For this dataset, I will set a threshold of 30% to the percentage of missing values.

  • Any columns with 30% or more missing values will be removed from the DataFrame.
  • Columns with less than 1% of missing values will have those particular rows removed.
  • All other columns with missing values will be looked at individually to decide how to fill the missing data.

def missing_percentage():
    data_size = len(data) # length of dataframe
    missing = data.isnull().sum() # number of missing values per column
    missing_percentage = (missing/data_size)*100
    missing_percentage = missing_percentage[missing_percentage>0] # filter out columns with no missing values
    return missing_percentage

Training the Models¶

Before running the machine learning models, we will first use the LabelEncoder to encode non-numeric columns into numeric.We will split the dataset into a 3:7 ratio of test and train set.

SMOTE and random under-sampling will be applied to the minority and majority class of the training set because it is imbalanced. Usually, a balanced dataset will use the accuracy score as a measure of performance for the models. However, since this dataset is heavily imbalanced, we will use the f1 score to measure model performance instead.

Model Selection¶

models = [KNeighborsClassifier(), DecisionTreeClassifier(), RandomForestClassifier(), LogisticRegression()]
table = pd.DataFrame(columns = ['Model', 'Precision', 'Recall', 'F1 Score'])
best_feature = []
cur_score = 0
index = 0
for model in models:
    cur_model = model
    model_name = model.__class__.__name__
    model_fit = cur_model.fit(X_train, Y_train)
    model_predict = model_fit.predict(X_test)
    precision = precision_score(Y_test, model_predict)
    recall = recall_score(Y_test, model_predict)
    f1 = f1_score(Y_test, model_predict)   
    # update higest f1 score and model features
    if (f1 > cur_score) and (model_name!='KNeighborsClassifier'):
        cur_score = f1
        best_feature = cur_model.feature_importances_
    # fill current row
    table.loc[index] = model_name, precision, recall, f1
    index += 1

Feature Importances¶

Observing the graph, we can see that features that describes general information about the borrower have no impact in deciding whether the credit card transaction will be fraud or not. Examples of such features includes:

  • the contract type
  • gender of the borrower
  • whether the borrower owns a car
  • if they have any children

On the other hand, below are a list of the top features in deicding if a credit card transaction is fraud.

  • the product combination
  • the term of previous credit at application of the previous application
  • the selling area of the seller's place
  • the channel type that the client's previous application was acquired on
  • the portfolio of the previous application(cash, pos, car, etc)

Conclusion and Next Steps¶

Using SMOTE and undersampling methods, we can reduce the impact of imbalanced data on our training models.

After comparing four different binary classification models(K-Neighbors, Decision Tree, Random Forest and Logistic Regression), we found that the model that performed the best amongst the four models is the Random Forest Classifier with an F1 score of 0.8.

We found out that general information on a client does not tell much about whether the transaction could be fraud or not. On the other hand, information related to the client's previous applications and the seller area are the best features in identifying a transaction's fraudulence.

To further improve the model, we could use Grid Search to find the Random Forest Classifier's most optimal parameters that will enhance the current model's performance.
